Thank you for installing {{ .Chart.Name }} ({{ include "open-webui.image" . }}).

Release name: {{ .Release.Name }}
Namespace:    {{ .Release.Namespace }}

1. Wait for Open WebUI to become ready:

   kubectl -n {{ .Release.Namespace }} rollout status deploy/{{ include "open-webui.fullname" . }}

2. Reach Open WebUI:
{{- if .Values.ingress.enabled }}
{{- range .Values.ingress.hosts }}
   http{{ if $.Values.ingress.tls }}s{{ end }}://{{ .host }}
{{- end }}
{{- else }}
   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "open-webui.fullname" . }} 8080:{{ .Values.service.httpPort }}
{{- end }}

3. This chart is a pure consumer: the database connection and OAuth credentials are
   supplied explicitly via .Values.env (wire DATABASE_URL from the provisioned
   Secret's `url` key, and OAUTH_CLIENT_ID/SECRET from the SSO Secret). The database
   and SSO app are provisioned separately from .Values.databases / .Values.sso.
